# GameNight OS Game Plan

The MVP framework is BUILT (original Phases 0-5: scaffold, auth+crew, schedule, bracket engine, TV broadcast, recap card). This document now tracks the features era. BACKLOG.md holds the full idea list; this file holds what's actively being built and the standing milestone.

## Standing milestone: MVP night
A real friend group runs one full game night through the app: RSVPs collected, bracket run live, TV view on the big screen, recap card in the group chat.

Pre-night checklist (email goes live here):
- Buy a cheap domain (~$10/yr, Porkbun/Cloudflare/Namecheap). App stays on replit.dev; domain is only for email.
- Verify the domain in Resend (add the DNS records they give you).
- Switch the from-address in apps/server/src/email.ts from onboarding@resend.dev to the new domain.
- Test: send a magic link to an email that is NOT the Resend account email.
- (Softened by password signup: friends can create accounts with zero email involvement.)

## NOW: Beerio Kart game pack
A 1:1 replica of the existing Beerio Kart Bracket app (github.com/knightjames1998/Beerio-Kart-Bracket), living inside GameNight OS as the first real game pack. Visually and functionally identical to the original; the ONE difference is that it binds to GameNight identity so results feed lifetime stats.

Decided:
- Results sync to lifetime stats (decision logged in BACKLOG.md).
- Everything else stays 1:1: double elim with grand final reset, Grand Prix mode, racer colors, its look and feel.

Standing rule (applies to this and every future game mode): a TV/spectator view ships with the mode. Beerio Kart's is its original live-QR spectator flow, already part of the 1:1 port.

Port plan (expect 1-2 full sessions):
1. Session A (BUILT, awaiting James verification): the replica: port the ~2,270-line App.tsx as a self-contained page at its own route, launched via a "Beerio Kart" button next to "Start bracket" on events. Its state (full serialized bracket/GP session) persists to a new beerio_sessions + beerio_hof tables (opaque jsonb state, original API contracts reimplemented in apps/server/src/beerio.ts), replacing the original app's session-sync backend with our own API + WebSocket hub. Schema change: ships with pnpm db:push.
2. Session B (BUILT, apply after A): the stats binding: entrants are picked from crew members instead of free-typed names (guests allowed but excluded from stats). On completion, final standings materialize into the reserved matches/match_participants tables, the first real use of those. This is the foundation the future cross-game Legacy stats read from.

Open questions to answer during the port (not before):
- Whether the original's fonts/assets can be vendored cleanly or need substitutes.
- How GP mode standings map onto match_participants (placement column exists for exactly this; verify it fits).

## Standing deployment notes
- Schema changes ship with the exact command: pnpm db:push (runs drizzle-kit push against DATABASE_URL).
- WebSocket hub is in-process memory: SINGLE INSTANCE ONLY on any deployment.
- Replit env vars: DATABASE_URL (built-in Postgres), RESEND_API_KEY.
